The "checkbox hack" is a CSS technique that uses a hidden checkbox to maintain state (checked/unchecked). By styling the label and using the :checked pseudo-class, you can create interactive components like accordions, tabs, and menus without any JavaScript.

This works because checkboxes can be controlled via labels, and CSS can detect their checked state using the :checked selector. The sibling selectors (+ and ~) allow you to style elements based on the checkbox state.

CSS cannot smoothly animate height: auto because "auto" isn't a numeric value. Instead, we use max-height with specific pixel values:

  • Closed state: max-height: 0 with overflow: hidden
  • Open state: max-height: 1000px (or any value larger than the content)

The browser can smoothly transition between these numeric values. The key is making the max-height large enough to fit all content, but not so large that the animation appears slow.

If you want only one item open at a time (exclusive accordion), you would use radio buttons instead of checkboxes, since radio buttons in the same group automatically deselect when another is selected.

CSS provides two sibling selectors that are essential for the checkbox hack:

  • Adjacent sibling (+): Selects the immediately next sibling. Example: .checkbox:checked + .label targets the label right after the checkbox.
  • General sibling (~): Selects all siblings that come after. Example: .checkbox:checked ~ .content targets the content div even if there are other elements in between.

These selectors only work for siblings (elements with the same parent) and only target elements that come after the first element, not before.

While this pure CSS accordion works visually, it has accessibility limitations. Screen readers will announce it as a series of checkboxes and labels, not as an accordion.

For better accessibility, you should add ARIA attributes:

  • role="button" on the label
  • aria-expanded="true/false" to indicate open/closed state
  • aria-controls="content-id" to link header to content

However, these ARIA attributes require JavaScript to update dynamically, so for production accordions with full accessibility, a small amount of JavaScript is recommended.

The pure CSS accordion is excellent for:

  • Simple FAQ sections where JavaScript isn't available or desired
  • Static sites or sites with minimal JavaScript
  • Learning and understanding CSS capabilities
  • Progressive enhancement (works without JavaScript)

For complex requirements (analytics tracking, keyboard navigation, advanced animations, or better accessibility), consider using JavaScript to enhance the functionality.

How This Works

This accordion uses three key CSS techniques:

  • Hidden checkboxes: Store open/closed state without being visible
  • :checked pseudo-class: Detect when checkboxes are toggled
  • Sibling selectors (+, ~): Style content based on checkbox state

The header is a <label> connected to the checkbox via the for attribute. Clicking the label toggles the checkbox, which triggers CSS changes on sibling elements.

No JavaScript required! This demonstrates the power of modern CSS for creating interactive components.
