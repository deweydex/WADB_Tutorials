Getting Started with CSS Tabs

Pure CSS tabs use the :target pseudo-class to show and hide content based on the URL hash. When you click a tab link, the URL changes (e.g., to #tab1), and the panel with that ID becomes active.

How It Works

  1. Each tab is a link (<a href="#tab1">) that changes the URL hash
  2. Each content panel has a matching ID (<div id="tab1">)
  3. CSS hides all panels by default (display: none)
  4. The :target selector shows the panel matching the current hash
  5. The first panel shows by default when there's no hash

This technique requires no JavaScript, making it perfect for static sites, progressive enhancement, and situations where you want to minimize dependencies.

Features & Benefits

CSS-only tabs offer several advantages for certain use cases:

Built-in Browser Support

  • Back/Forward buttons work: Browser history navigation switches tabs automatically
  • Shareable links: You can link directly to a specific tab (e.g., page.html#tab2)
  • Bookmark support: Users can bookmark specific tabs
  • No dependencies: Works without JavaScript libraries or frameworks

Performance Benefits

  • No JavaScript parsing or execution overhead
  • Instant tab switching (no DOM manipulation)
  • Works even if JavaScript fails to load or is disabled
  • Smaller page size without JS dependencies

Use Cases

Pure CSS tabs work great for:

  • Documentation sites with multiple sections
  • Product pages showing different features or specs
  • Portfolio sections (About, Projects, Contact)
  • Settings panels with categorized options
  • Progressive enhancement (works without JS, enhanced with JS)

Code Examples

Here's the essential code structure for CSS-only tabs:

HTML Structure

<div class="tabs"> <nav class="tab-nav"> <a href="#tab1">Tab 1</a> <a href="#tab2">Tab 2</a> </nav> <div class="tab-content"> <div class="tab-panel" id="tab1"> Content for tab 1 </div> <div class="tab-panel" id="tab2"> Content for tab 2 </div> </div> </div>

Essential CSS

/* Hide all panels by default */ .tab-panel { display: none; } /* Show the targeted panel */ .tab-panel:target { display: block; } /* Show first panel if no hash */ .tab-panel:first-of-type { display: block; } /* Hide first panel when another is targeted */ .tab-panel:target ~ .tab-panel:first-of-type { display: none; }

Adding Animations

@keyframes fadeIn { from { opacity: 0; transform: translateY(10px); } to { opacity: 1; transform: translateY(0); } } .tab-panel { animation: fadeIn 0.5s ease; }

Best Practices & Limitations

Accessibility Considerations

While functional, pure CSS tabs have accessibility limitations:

  • Screen readers don't announce tab states (active/inactive)
  • No ARIA attributes without JavaScript
  • Keyboard navigation works (Tab key) but isn't optimal (no arrow keys)
  • Consider adding ARIA roles with minimal JavaScript for better accessibility

When to Use JavaScript Instead

Add JavaScript if you need:

  • Full ARIA support for screen readers
  • Arrow key navigation between tabs
  • Analytics tracking of tab switches
  • Complex animations or transitions
  • Programmatic tab switching
  • Preventing certain tabs from being accessible until conditions are met

Progressive Enhancement

The best approach is often progressive enhancement:

  1. Start with CSS: Tabs work without JavaScript
  2. Add JavaScript: Enhance with ARIA, keyboard nav, analytics
  3. Maintain fallback: Ensure tabs still work if JS fails

This tabbed interface uses the :target pseudo-class to detect which tab should be active based on the URL hash:
